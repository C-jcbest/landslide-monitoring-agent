# 功能需求

## 1. 用户认证与账号

- 支持本地账号登录、登出和访问令牌刷新，只有拥有北斗监测平台上游账号密码凭证时才能够获取监测数据。
- 第一阶段仅支持本地账号认证，不接入 OAuth、OIDC、LDAP、企业微信、钉钉或其他外部身份源。
- 支持用户查看和维护自己的昵称、邮箱等资料。
- 支持用户查看自己的北斗凭据绑定状态，允许普通用户自行录入上游账号密码。
- 北斗凭据绑定前必须验证上游可用性，保存时必须加密。
- 所有需要访问北斗/GNSS 上游数据的用户都必须单独绑定北斗平台凭据。

## 2. 对话工作台

- 支持历史消息分页加载、重新生成。
- 支持用户选择可用模型。
- 支持流式输出文本、分析进度、工具进度、结构化面板、下一步建议和错误状态。
- 支持用户取消生成。
- 支持中断恢复，包括业务确认、工具审批和长任务恢复。
- 支持复制消息、折叠长消息、点击后续动作继续追问。

## 3. 结构化面板与图表

- 支持表格、信息卡、指标摘要、趋势图、异常卡、时间线、确认卡、审批卡、站点对比和异常排名。
- 前端优先采用成熟组件库实现基础 UI 和业务组件。
- 生成式 UI 必须基于开发者预定义的组件 catalog、registry 和 props schema；模型只能生成结构化 UI spec、面板数据或受控动作标识，不能生成可执行前端代码。
- 图表应支持缩放、框选、点选、重置和保存图片。
- 图表点选或框选后可生成下一轮追问上下文。
- 大表只展示预览；完整数据通过受控缓存或分页恢复。
- 面板和 artifact 必须经过白名单和 schema 校验，不能执行模型生成代码。
- 第一阶段生成式 UI 优先服务于结构化面板选择、面板组合和组件 props 生成；复杂自由布局生成、外部组件加载和用户自定义组件市场延后。

## 4. 北斗/GNSS 数据能力

- 支持按当前用户绑定的北斗凭据登录、会话缓存和过期重登，过期重登要求用户无感知自动进行。
- 不支持全局北斗业务账号兜底；缺少用户级凭据时应返回可理解的授权缺失提示。
- 支持监测点分组、列表和详情查询。
- 第一阶段站点详情展示经度、纬度、海拔等坐标信息；不内置完整 GIS 地图能力。
- 支持 GNSS 实时数据和日监测数据查询。
- 支持按站点名称、编码、缩写、别名、模糊名称、不准确表达或上下文指代进行语义实体解析；当候选不唯一时，应向用户澄清，而不是用固定关键词规则武断选择。
- 支持按时间范围、数据类型、采样参数和粒度查询。
- 常规查询优先使用适合分析的较粗粒度；用户明确要求实时或分钟级时再查询高频数据。
- 支持站点类型能力门禁，不适合 RTK/PJK 异常分析的对象必须拦截并说明。
- 第一阶段不做地图点位聚合、地图选点、轨迹回放或空间分析；如需地图外链或复制坐标，应避免引入地图 SDK 和地图密钥。
- 支持自然语言生成监测数据分析报告，可按单个监测点、多个监测点、分组进行，输出pdf文件并能够提供给用户下载
- 支持监测数据分析，并根据分析报告模板回复给用户，此部分与分析报告文件有区别，此功能为正常的对话chat内容

## 5. 异常候选分析

- 支持对 GNSS 位移序列生成异常候选证据。
- 第一阶段采用规则型、可解释、可复核的异常候选检测器，不引入训练型 ML 模型或复杂训练流程。
- 覆盖突变、阶跃、漂移、噪声突增、多轴同步、垂向突出、水平位移和数据缺口等场景。
- 自动候选不能直接决定最终异常结论；最终答复必须结合证据、限制和业务规则。

## 6. 智能体能力

- 支持自然语言理解、多步工具调用、参数追问、结果总结和后续动作推荐。
- 需要理解用户意图、模糊实体、缩写、别名、不准确输入或上下文省略的流程，优先由智能体或语义理解模块参与
- 第一阶段模型运行时从数据库中的启用模型配置读取；支持 OpenAI-compatible Chat Completions 风格 provider 作为优先接入形态。
- 支持短期记忆、长期记忆、用户偏好和会话上下文。
- 支持工具错误恢复和用户可理解的失败说明。
- 支持 Human-in-the-Loop 审批敏感工具调用。
- 支持业务确认，例如大分页、参数缺失、订阅副作用操作。
- 支持可选 MCP 工具接入，并限制启用范围。

## 7. 邮件订阅

- 支持按日、按周和固定间隔订阅。
- 支持自然语言进行创建、列表、详情等操作，以及网页执行暂停、恢复、删除、立即运行和运行历史操作。
- 自然语言订阅管理必须转成结构化操作。
- 创建、更新、删除、暂停、恢复和立即运行属于副作用操作，必须先确认。
- 收件邮箱来自用户资料，不从聊天文本临时解析。
- 调度器必须使用数据库锁或等价机制避免多实例重复执行。
- 邮件内容由智能体生成，支持纯文本和 HTML 渲染。



# 领域规则

## 坐标与指标

- N / north：北向坐标或北向位移。

- E / east：东向坐标或东向位移。

- U / up：高程或垂向坐标/位移。

- H / horizontal：水平合成位移，`sqrt(N^2 + E^2)`。

- D3 / three_d：三维合成位移，`sqrt(N^2 + E^2 + U^2)`。

- residual：相对基准坐标的位移差，面向用户通常以毫米表达。
