# 001 UI 样式优化与确认弹窗重构

## 背景

当前 Landslide Monitoring Agent 前端页面采用深色暗黑风格，包含部分半透明玻璃态样式，某些交互上使用了浏览器默认的 `confirm` 弹窗。此外，还存在切换会话时列表抖动、边框线参差不齐等视觉小缺陷。
为了提升产品的专业感和美观度，需要将前端整体重构为简约高级的白色调风格（类似于 ChatGPT），取消上方菜单栏内容与清空历史按钮（保留菜单栏结构但清空其内部内容），并在前端实现一套美观的自定义 UI 弹窗组件替换浏览器默认弹窗，以极大提升用户观感。同时，将相关交互规范补充到 Agent 规则中。此外，为了优化宽屏体验，需要支持左侧会话列表区域折叠功能，并适当缩小侧边栏的宽度。

## 目标

1. **规则补充与弹窗重构**：
   - 将“删除会话时弹窗禁止使用默认浏览器信息弹窗，使用 UI 组件，避免前端样式不美观”这句描述精简优化为：“**禁止使用浏览器默认弹窗（如 `confirm`/`alert`），应使用统一的 UI 组件（Modal）进行交互确认，以确保界面风格一致和美观。**”，并将其补充到 `AGENTS.md` 的“本项目的十诫”与“应避免的常见陷阱”中。
   - 在前端开发通用的 UI `Modal` 组件，用于删除会话确认、退出登录确认等。
2. **解决切换会话列表抖动问题**：
   - 优化 `Sidebar.tsx` 中会话列表块（Session Item）的样式，消除切换会话时高度或边框变动带来的抖动，保证布局稳定。
3. **边框线对齐与调整**：
   - 调整左侧菜单栏、上方菜单栏、主聊天区域及输入框区域等边框的线条位置与样式，使其对齐，避免重叠或参差不齐。
4. **白色调简约高级风格重构（ChatGPT 风格）**：
   - 重构全局 index.css 变量与 Tailwind 样式。
   - 主聊天背景改为白色/浅灰（例如 `#ffffff`），侧边栏背景改为极浅灰色（例如 `#f9f9f9`），文字主色采用深灰/黑（如 `#1f2937`）。
   - 适配 Markdown 样式的文本、代码块和表格颜色为浅色主题。
   - 登录注册页面（AuthScreen）进行样式适配，统一为 ChatGPT 浅色调简约风格。
5. **侧边栏折叠与宽度微调**：
   - 将左侧会话列表（Sidebar）区域宽度微调（由原 `w-80` 缩小为更紧凑的 `w-64` 或 `w-72`）。
   - 参考 ChatGPT 样式，增加侧边栏折叠与展开的交互。在侧边栏折叠时，侧边栏滑出或隐藏，右侧聊天窗口自适应填满，并在界面上提供一个精致的“展开侧边栏”悬浮按钮；侧边栏展开时，侧边栏正常显示，并提供“折叠侧边栏”按钮。
6. **个人信息下拉菜单与功能弹窗**：
   - 在左下角个人信息区增加点击触发的下拉菜单（Popover/Dropdown）。
   - 菜单项包含：**设置**、**报表记录**、**订阅历史**、**退出登录**。
   - 点击“设置”、“报表记录”、“订阅历史”后，分别以高级极简风格的 UI 弹窗形式展示对应的内容（可先提供 Mock 表单/展示）。
   - 点击“退出登录”后，以自定义确认弹窗提示，确认后再执行注销。
7. **优化输入框样式**：
   - 重新设计输入框，宽度限制在主聊天区居中（max-w-4xl），外围带有轻微的卡片阴影（类似 ChatGPT 的阴影风格）。
   - 输入框内部左侧留出未来扩展按钮的位置，右侧是更精致的发送按钮，提升未来功能扩展性。
8. **取消上方菜单栏内容及按钮**：
   - 移除 `ChatWindow.tsx` 中的上方菜单栏（Top Header）内部的所有文本内容、状态标识与清空历史按钮，仅保留一个空白的 Header 条。

## 非目标

1. **真实后端设置/订阅交互**：
   - 本次不做真实的“报表记录”和“订阅历史”后端接口，弹窗内容以符合设计风格的静态 Mock 或前端交互展示为主。

## 技术方案

1. **规则补充**：
   - 修改 `AGENTS.md`，将“十诫”扩充为十一诫，增加第十一诫：“**禁止使用浏览器默认弹窗（如 `confirm`/`alert`），应使用统一的 UI 组件（Modal）进行交互确认，以确保界面风格一致和美观。**”，并在“应避免的常见陷阱”里加上一条“❌ 使用浏览器原生的 alert/confirm 等弹窗”。
2. **样式重构 (白色调 & ChatGPT 风格)**：
   - 重新定义 `frontend/src/styles/index.css` 的 `--background`、`--foreground`、`--border` 等变量。
   - 调整 `.glass`、`.glass-card`、`.glass-input` 等样式，使其融合在白色简约背景中，或者用现代阴影和纯色替代之。
   - 对 MarkdownMessage 中的标题、正文、`code`、`pre` 标签颜色进行全面漂白（浅色背景色、深色字）。
3. **侧边栏折叠与缩窄**：
   - 将 Sidebar 的 className 宽度改为 `w-64` 或 `w-72`。
   - 在 `App.tsx` 中引入侧边栏折叠状态 `isSidebarCollapsed`。
   - 如果侧边栏处于折叠状态，将 Sidebar 隐藏，同时在聊天区域的左侧上方悬浮渲染一个“展开侧边栏”的微缩按钮。
   - 如果展开，Sidebar 的顶部 Header 或空白顶部加一个折叠侧边栏的按钮（ChatGPT 经典的面板折叠图标，如双箭头或带有竖线的控制图标 `SidebarCollapse` 等，也可以使用 `ChevronLeft`）。
4. **消除抖动与对齐边框**：
   - 设定 Sidebar 中每个 Session 按钮为固定的 `h-11`，统一 border 宽度，移除 hover 或 active 引起的 layout shift。
   - 采用全局的 1px 细线 `border-gray-200` 或 `border-neutral-200` 用于分界。
5. **自定义 UI Modal 与 Dropdown**：
   - 在前端编写 `Modal` 组件（支持 `title`、`onClose`、`children`、`footer`）。
   - 在前端编写 `DropdownMenu` 组件，或者在 `Sidebar` 底部使用 React `state` 控制的绝对定位菜单。
   - 实现“删除确认弹窗”、“退出登录确认弹窗”，以及“设置”、“报表记录”、“订阅历史”弹窗。
6. **输入框优化**：
   - 将 input 的 outer container 设置为 `shadow-lg` 且圆角，背景为白色，限制最大宽度，并居中。
   - 添加左侧 placeholder 扩展按钮。

## 验收

- [ ] `AGENTS.md` 成功补充相关开发规范描述。
- [ ] 切换会话列表时没有任何抖动，列表项高度及文字排版稳定。
- [ ] 界面整体转为 ChatGPT 风格的白色/浅灰色系，主聊天背景为纯白，侧边栏为浅灰，字体易读，边框对齐且不参差不齐。
- [ ] 侧边栏宽度已缩窄，且可以进行折叠与展开。折叠后聊天区域能够自动占满，界面上能够通过折叠/展开图标顺畅触发侧边栏隐藏/显现。
- [ ] 登录注册页面（AuthScreen）成功适配白色简约风格。
- [ ] 删除会话时无原生 `confirm` 弹窗，改为美观的自定义确认 Modal；确认删除后，会话能成功被删除。

- [ ] 输入框变成 ChatGPT 风格的圆角阴影栏，居中显示，包含未来扩展按钮占位，支持正常发送消息。
- [ ] `ChatWindow.tsx` 取消了上方菜单栏（Top Header）内容与清空监测历史按钮，仅保留一个与左边栏、右聊天区完美对齐的空白顶部边框栏，UI 清爽。
