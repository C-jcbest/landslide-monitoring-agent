# 001 前端页面开发 — 测试计划

## Happy Path (正常使用场景)

1. **用户注册与登录 (Auth Flow)**
   - 输入合法的邮箱和强密码，点击注册 -> 后端成功返回 Token -> 本地成功存入 `localStorage` 且自动跳转至聊天界面。
   - 输入正确的账号密码点击登录 -> 后端返回 Token -> 跳转页面。
2. **会话列表增删改查 (Session CRUD)**
   - 登录后，侧边栏正确渲染该用户的所有会话列表。
   - 点击“新建会话” -> 成功请求后端，创建新 Session 并在侧边栏顶部生成新的会话项。
   - 点击会话名旁边的“编辑/重命名” -> 弹出重命名 Dialog，输入新名字保存 -> 侧边栏和头部显示新名称。
   - 点击“删除会话” -> 发起 API 请求成功，会话从侧边栏消失；如果是当前选中会话，则自动切换到剩余的最优会话或空白状态。
3. **消息接收与流式输出 (Streaming Chat)**
   - 在输入框键入问题并回车 -> 发送后输入框清空，消息展示在对话流中。
   - 对接流式 SSE 接口 `/chatbot/chat/stream`，解码器流式追加字符，直到收到 `done: true`。
   - 使用 `react-markdown` 正确解析 Agent 回复中的粗体、代码块、列表和表格，样式优雅。
4. **人机交互中断与恢复 (Human-in-the-loop)**
   - 发送触发敏感操作的请求，Agent 返回 `ask_human` 提问并挂起 -> 流式输出完毕后，前端主输入框被禁用，聊天区域内展现一个高亮的 `HumanPromptCard` 中断输入提示卡片。
   - 用户在卡片内键入答复并提交 -> 答复内容被作为新消息提交给对话流，重新启动 SSE 连接，对话界面成功输出 Agent 恢复运行后的新一轮回答，同时卡片隐藏，主输入框恢复可用。

## 边界场景 (Boundary Scenarios)

1. **Token 过期与未登录劫持**
   - 尝试直接访问聊天界面（无 `user_token`）-> 应用捕获拦截，自动跳转到 `/login` 登录页。
   - 调用接口时，若后端返回 `401 Unauthorized`（Token 已过期/被篡改）-> 前端清空 `localStorage` 缓存的 Token，并强制重定向回登录页。
2. **空输入与过长输入**
   - 输入框为空或仅有空格时，发送按钮禁用，回车无效。
   - 输入字符超过限制时，表单提示超长警告，无法提交。
3. **无会话状态**
   - 新注册用户第一次登录后，侧边栏为空，聊天主界面展示欢迎画面并引导用户“点击新建会话”开始聊天，输入框和发送按钮暂时禁用。

## 失败场景 (Failure Scenarios)

1. **认证失败**
   - 注册时输入已被占用的邮箱 -> 后端返回 `400` -> 前端错误横幅提示“该邮箱已被注册”。
   - 登录时输入错误的密码 -> 后端返回 `401` -> 界面提示“邮箱或密码不正确”。
2. **后端服务断开 / 网络故障**
   - 在流式传输对话时突然网络断开 -> 流式读取器捕获异常 -> 对话气泡下方显示红色的“网络连接中断，请重试”错误提示，发送按钮重新启用。
   - 创建会话请求失败 -> 弹出 Toast 提示“创建会话失败，请检查网络”。

## 测试层级、数据与依赖

1. **前端单元/集成测试 (Unit & Component Test)**：
   - 采用 **Vitest** + **React Testing Library**。
   - **Mocking**：Mock 全局 `window.fetch`，模拟 HTTP 响应（200、400、401）和 SSE 流式输出。
   - 关键断言：
     - `AuthScreen`：验证表单合法性校验、错误提示是否展示。
     - `fetchStream` 辅助函数：输入模拟流，验证最终拼装的文本以及是否能正确触发完成回调。
2. **端到端集成验证 (Manual E2E Test)**：
   - 启动本地 FastAPI 后端服务（运行于 `http://localhost:8000`）。
   - 启动 Vite 前端服务（运行于 `http://localhost:5173`）。
   - 开发者根据“Happy Path”进行完整流程操作，并用 Chrome DevTools 检查：
     - Network 中各请求的 Request/Response 结构。
     - LocalStorage 中的 Token。
     - 触发中断时，Console 是否打印出对应的状态。
