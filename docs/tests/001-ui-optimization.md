# 001 UI 样式优化与确认弹窗重构 — 测试计划

## Happy Path

### 1. 样式与主题表现 (ChatGPT 浅色主题)
- **测试步骤**：
  1. 启动前端项目，进入页面。
  2. 验证整个聊天窗口背景为白色 (`bg-white` 或极浅灰 `#ffffff`)，侧边栏背景为淡灰色 (`bg-gray-50`/`bg-[#f9f9f9]`)，文字以深灰色 (`text-gray-800` / `#1f2937`) 呈现。
  3. 验证聊天流中的 Markdown 渲染效果，代码块 (`code`/`pre`) 呈现浅色底色 (`bg-gray-100` 或 `bg-neutral-100`)，标题和正文清晰可读。
  4. 验证登录/注册页面 (`AuthScreen.tsx`) 的白色极简风格是否适配成功。
- **关键断言**：页面无深蓝色玻璃渐变背景，全站界面呈白色调，边框分界线对齐无重叠。

### 2. 会话切换稳定性
- **测试步骤**：
  1. 在左侧会话列表中拥有 2 个以上会话。
  2. 连续快速点击切换不同的会话。
- **关键断言**：在切换瞬间，列表项的高宽不发生任何微小的位移或拉伸，整个列表及右侧聊天区无抖动。

### 3. 侧边栏折叠与展开功能
- **测试步骤**：
  1. 验证侧边栏已缩窄至紧凑布局（宽度由原 `w-80` 调整为 `w-64` 或 `w-72`）。
  2. 点击侧边栏顶部 Header 右侧的“折叠侧边栏”按钮。
  3. 确认侧边栏被成功折叠/隐藏，右侧聊天区域横向自适应填满。
  4. 确认聊天区域左上角出现一个精致的“展开侧边栏”按钮（双折线或侧栏展开图标）。
  5. 点击该展开按钮，确认侧边栏能够平滑地重新显现。
- **关键断言**：折叠和展开状态切换顺畅，不引发任何布局重叠或功能失效。

### 4. 自定义删除会话弹窗
- **测试步骤**：
  1. 悬停在某个会话块上，点击删除按钮。
  2. 验证页面是否弹出美观的自定义对话框 Modal，而非浏览器的原生 `confirm` 弹窗。
  3. 点击自定义 Modal 中的“取消”按钮，验证弹窗关闭，会话未被删除。
  4. 再次点击删除，点击自定义 Modal 中的“确认”按钮。
- **关键断言**：弹窗成功关闭，且对应的会话从列表中消失，后端删除 API 成功触发。

### 5. 个人信息下拉菜单与功能弹窗
- **测试步骤**：
  1. 点击侧边栏左下角的个人信息区域。
  2. 验证弹出下拉菜单 (包含“设置”、“报表记录”、“订阅历史”、“退出登录”)。
  3. 分别点击“设置”、“报表记录”、“订阅历史”。
  4. 验证各自在屏幕中央弹起极简浅色调的自定义 Modal 弹窗。
  5. 点击 Modal 的关闭 `X` 按钮，验证弹窗关闭。
- **关键断言**：下拉菜单能正确弹出与收起，各项 Modal 点击均能正常呈现和关闭。

### 6. 退出登录确认弹窗
- **测试步骤**：
  1. 点击个人信息区域，选择“退出登录”。
  2. 验证页面弹出自定义的退出确认弹窗。
  3. 点击“取消”，验证停留在当前页面。
  4. 再次点击“退出登录”，点击 Modal 中的“确定”。
- **关键断言**：点击确定后，本地 Token 被清除，页面重定向并返回到简约白底的登录页面。

### 7. 输入框优化
- **测试步骤**：
  1. 验证输入框区域呈圆角卡片，限制宽度并水平居中，且底部有柔和阴影。
  2. 输入框左侧包含扩展按钮（+ 或纸夹）占位，右侧发送按钮样式精致。
  3. 输入文字并点击发送，或者敲击回车发送。
- **关键断言**：消息能正常发送，且无样式塌陷。

### 8. 上方菜单栏留白
- **测试步骤**：
  1. 进入活跃会话。
  2. 查看页面顶部区域。
- **关键断言**：上方菜单栏保留但内容完全清空（没有会话名、会话 ID、清空历史按钮），仅仅作为 1px 的整洁顶部边界线（在折叠时渲染侧边栏展开图标）。

## 边界场景

1. **空会话列表**：
   - **步骤**：当用户删除所有会话或刚注册新账号时，验证侧边栏是否正常展示“暂无历史监测会话”的浅色调极简占位符，排版无位移。
2. **超长会话重命名**：
   - **步骤**：重命名会话为超长文本，在输入和展示状态下，验证是否被正常截断 (`truncate`)，且没有把右侧操作按钮挤出可视范围或导致列表宽度变化。
3. **点击外部区域收起下拉菜单**：
   - **步骤**：打开个人信息下拉菜单后，点击屏幕任意其他区域（如聊天区）。
   - **关键断言**：下拉菜单能够自动收起。
4. **弹窗背景滚动锁定**：
   - **步骤**：在聊天消息较多（页面有滚动条）时弹出 Modal，尝试在 Modal 外部滚动。
   - **关键断言**：主页面背景被锁定不可滚动。

---

## 失败场景

1. **输入框禁用状态**：
   - **步骤**：在 Agent 正在生成回复时（loading 态），验证输入区是否呈灰色禁用状态，且无法输入和提交。
2. **删除会话请求失败**：
   - **步骤**：模拟网络异常（可让 API 接口返回 500），点击删除会话并确认。
   - **关键断言**：弹出错误提示（`alert` 或者是错误 Modal），且 Modal 能妥善处理，不发生前端白屏崩溃。

## 测试层级、数据与依赖

1. **单元测试 (Unit Test)**：
   - 我们将针对新增的 `Modal` 组件编写 React Unit Test，确保其能响应 `onClose`、`children`、以及 `footer` 按钮 of the click 事件。
   - 单元测试运行命令：`npm run test`。
2. **手动集成与冒烟测试 (Manual Test)**：
   - 依赖本地 FastAPI 后端与前端 Vite 服务器的联调。
   - 用户与开发者共同在 Chrome 浏览器中按照“Happy Path”步骤进行逐项对照勾选。
